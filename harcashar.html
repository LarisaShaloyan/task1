That is javascript(Ինչ է ջավասկրիպտը:)

JavaScript-ը բազմակողմանի ծրագրավորման լեզու է, որը հիմնականում օգտագործվում է ինտերակտիվ վեբ էջեր և վեբ հավելվածներ ստեղծելու համար: Այն ստեղծվել է Բրենդան Էյխի կողմից, երբ նա աշխատում էր Netscape Communications Corporation-ում և առաջին անգամ թողարկվեց 1995 թվականին: Այդ ժամանակից ի վեր այն դարձել է ծրագրավորման ամենատարածված լեզուներից մեկը, հատկապես վեբ մշակման համար: JavaScript-ը բարձր մակարդակի, մեկնաբանվող(ինտերպրետացվող) լեզու է, որն իրականացվում է վեբ բրաուզերների կողմից՝ կայքերում դինամիկ գործառույթներ ապահովելու համար: Սա Համաշխարհային ցանցի հիմնական տեխնոլոգիան է՝ HTML-ի և CSS-ի հետ միասին:

What are the differences between Javascript and Java (Որո՞նք են տարբերությունները Javascript-ի և Java-ի միջև:)

JavaScript-ը և Java-ն ծրագրավորման երկու տարբեր լեզուներ են՝ տարբեր ծագումներով, դիզայնի փիլիսոփայությամբ և օգտագործման դեպքերով: Ահա մի քանի հիմնական տարբերություն JavaScript-ի և Java-ի միջև. Ծագումները: JavaScript. Մշակվել է Բրենդան Էյխի կողմից Netscape-ում 1995 թվականին: Սկզբնապես կոչվում էր «LiveScript», այն հետագայում վերանվանվեց JavaScript՝ Java-ի ժողովրդականությունը բարձրացնելու համար: Java: Մշակվել է Ջեյմս Գոսլինգի և նրա թիմի կողմից Sun Microsystems-ում (այժմ պատկանում է Oracle-ին) 1990-ականների սկզբին: Այն ի սկզբանե թողարկվել է 1995 թվականին։ Տիպ: JavaScript. Մեկնաբանված, դինամիկ մուտքագրված լեզու: Այն սովորաբար իրականացվում է վեբ բրաուզերներում և օգտագործվում է հաճախորդի կողմից սկրիպտավորման համար: Java. Կազմված, ստատիկ մուտքագրված լեզու: Այն կազմվում է բայթկոդի մեջ, որն այնուհետև գործարկվում է Java վիրտուալ մեքենայի (JVM) կողմից: Java-ն սովորաբար օգտագործվում է սերվերային հավելվածների, աշխատասեղանի հավելվածների և Android հավելվածների մշակման համար: Շարահյուսություն: JavaScript. Ունի C-ի նման շարահյուսություն և ազդում է այնպիսի լեզուներից, ինչպիսիք են C-ն և Java-ն, բայց այն նաև փոխառում է հասկացությունները ֆունկցիոնալ ծրագրավորման լեզուներից: Java. Ունի նաև C-ի նման շարահյուսություն և նախագծված է C և C++ ծրագրավորողների համար ծանոթ լինելու համար: Այն առարկայական կողմնորոշված լեզու է՝ ուժեղ ստատիկ մուտքագրմամբ: Հարթակ: JavaScript: Հիմնականում օգտագործվում է հաճախորդի կողմից վեբ մշակման համար: Այնուամենայնիվ, Node.js-ի գալուստով JavaScript-ը կարող է օգտագործվել նաև սերվերի կողմից մշակման համար: Java: Աշխատում է Java վիրտուալ մեքենայի վրա (JVM), որը թույլ է տալիս Java հավելվածներին անկախ լինել հարթակից: Java-ն սովորաբար օգտագործվում է ձեռնարկության մակարդակի հավելվածներ, սերվերի կողմի հավելվածներ և Android հավելվածներ ստեղծելու համար:

Explain the difference between "==" and "===" in Javascript.(Բացատրեք «=="-ի և «===»-ի տարբերությունը Javascript-ում:)

JavaScript-ում «==" և «==="-ը համեմատման օպերատորներ են, որոնք օգտագործվում են արժեքները համեմատելու համար: Այնուամենայնիվ, նրանք այլ կերպ են վարվում՝ պայմանավորված տիպային հարկադրանքով. "==" (Equality Operator - Loose Equality): «==" օպերատորը ստուգում է արժեքների հավասարությունը, բայց այն կատարում է տիպի պարտադրանք, եթե օպերանդները տարբեր տեսակի են: Եթե օպերանդները նույն տիպի են, ապա «=="-ն իրեն պահում է նույն կերպ, ինչ «===" (խիստ հավասարություն): Եթե օպերանդները տարբեր տեսակի են, JavaScript-ը փորձում է դրանք փոխարկել ընդհանուր տիպի, նախքան համեմատելը: Օրինակ՝ 1 == '1'-ը կվերադարձնի ճշմարիտ, քանի որ JavaScript-ը համեմատությունից առաջ փոխակերպում է «1» տողը թվի: "===" (Խիստ հավասարության օպերատոր): «===" օպերատորը ստուգում է արժեքների և տեսակների հավասարությունը՝ առանց որևէ տեսակի պարտադրանքի: Այն վերադարձնում է true, եթե օպերանդները նույն տեսակի են և ունեն նույն արժեքը։ Եթե օպերանդները տարբեր տեսակի են, «==="-ը վերադարձնում է false՝ առանց փոխակերպման փորձի:

console.log(1 == '1'); // true
console.log(1 === '1'); // false
console.log(0 == false); // true
console.log(0 === false);// false
console.log(null == undefined); // true
console.log(null === undefined); // false

harc 4

What are the different data types in javascript.(Որոնք են տվյալների տարբեր տեսակները Javascript-ում:)

JavaScript-ը դինամիկ տպագրված լեզու է, ինչը նշանակում է, որ փոփոխականները կարող են պահել ցանկացած տվյալների տիպի արժեքներ՝ առանց հստակ տիպի հայտարարագրերի: JavaScript-ն ունի ներկառուցված տվյալների մի քանի տեսակներ, որոնք կարելի է դասակարգել երկու հիմնական կատեգորիաների՝ պարզունակ տեսակներ և հղման տեսակներ: Պրիմիտիվ տվյալների տեսակները. number: Ներկայացնում է և՛ ամբողջ, և՛ լողացող թվերը: Օրինակ, 10, 3.14. string. ներկայացնում է նիշերի հաջորդականություն, որոնք փակցված են մեկ կամ կրկնակի չակերտներով: Օրինակ՝ «բարև», «աշխարհ»։ Boolean:: Ներկայացնում է ճիշտ կամ կեղծի տրամաբանական արժեքը: Օրինակ՝ ճշմարիտ, կեղծ։ Undefined: Ներկայացնում է փոփոխական, որը հայտարարվել է, բայց արժեք չի նշանակվել: Փոփոխականները ինքնաբերաբար մեկնարկվում են մինչև չսահմանված, երբ հայտարարվում են առանց հստակ արժեքի: Null: Ներկայացնում է որևէ օբյեկտի արժեքի միտումնավոր բացակայությունը: Այն հաճախ օգտագործվում է ցույց տալու համար, որ փոփոխականն իրեն վերագրված արժեք չունի: Խորհրդանիշ (ավելացվել է ECMAScript 6-ում). Ներկայացնում է եզակի նույնացուցիչ: Նշանները հիմնականում օգտագործվում են որպես օբյեկտների սեփականության բանալիներ՝ անունների բախումը կանխելու համար: Հղման տվյալների տեսակները. Օբյեկտ. ներկայացնում է բանալի-արժեք զույգերի հավաքածու, որտեղ ստեղները տողեր են (կամ Նշաններ) և արժեքները կարող են լինել ցանկացած տվյալների տիպի, ներառյալ այլ օբյեկտներ: Օբյեկտները օգտագործվում են տվյալների բարդ կառուցվածքները ներկայացնելու համար և հանդիսանում են JavaScript-ի նախատիպային ժառանգության հիմքը: Զանգված. ներկայացնում է տարրերի դասավորված հավաքածու, որոնցից յուրաքանչյուրը ճանաչվում է ինդեքսով: Զանգվածները կարող են պարունակել ցանկացած տվյալների տիպի տարրեր, ներառյալ այլ զանգվածներ և օբյեկտներ: Գործառույթ. ներկայացնում է կոդի բազմակի օգտագործման բլոկ, որը կարող է կանչվել զրոյական կամ ավելի արգումենտներով: Ֆունկցիաները JavaScript-ում առաջին կարգի քաղաքացիներ են, ինչը նշանակում է, որ դրանք կարող են վերագրվել փոփոխականներին, փոխանցվել որպես արգումենտ այլ գործառույթներին և վերադարձվել այլ գործառույթներից:

How do you declare variablies in javascript.(ինչպես եք հայտարարում փոփոխականները javascript-ում)

JavaScript-ում կարող եք փոփոխականներ հայտարարել՝ օգտագործելով երեք տարբեր հիմնաբառեր՝ var, let և const: Ահա թե ինչպես կարող եք օգտագործել դրանցից յուրաքանչյուրը. var: var-ը JavaScript-ում փոփոխականներ հայտարարելու ամենահին եղանակն է, և այն ունի գործառույթի շրջանակ: Var-ով հայտարարված փոփոխականները բարձրացվում են իրենց շրջանակի վերևում, ինչը նշանակում է, որ դրանք կարող են մուտք գործել նախքան դրանք հայտարարվելը: Այնուամենայնիվ, դրանք շրջափակված չեն, ինչը կարող է որոշ դեպքերում հանգեցնել անսպասելի վարքագծի, մասնավորապես՝ փակման դեպքում: let: Ներկայացված է ECMAScript 6-ում (ES6)
let-ը թույլ է տալիս հայտարարել փոփոխականներ, որոնք շրջափակված են:Let-ով հայտարարագրված փոփոխականները ունեն բլոկի մակարդակի շրջանակ, ինչը նշանակում է, որ դրանք հասանելի են միայն այն բլոկի ներսում, որտեղ նրանք հայտարարված են:
const: Նաև ներդրված է ES6-ում, const-ը թույլ է տալիս հայտարարել հաստատուններ, որոնց արժեքները չեն կարող վերանշանակվել սկզբնավորումից հետո:Const-ով հայտարարված փոփոխականները շրջափակված են let-ի պես, բայց դրանք պետք է սկզբնավորվեն արժեքով, և այդ արժեքը չի կարող փոխվել հետագայում կոդի մեջ: Այնուամենայնիվ, const-ով հայտարարված օբյեկտների և զանգվածների համար դրանց հատկությունները կամ տարրերը կարող են փոփոխվել, բայց փոփոխականն ինքնին չի կարող վերանշանակվել։ Ընդհանրապես խորհուրդ է տրվում օգտագործել const լռելյայն այն փոփոխականների համար, որոնք չեն վերանշանակվի, քանի որ դա օգնում է կանխել պատահական վերանշանակումները և կոդն ավելի ընթեռնելի է դարձնում: Օգտագործեք let այն փոփոխականների համար, որոնք պետք է վերանշանակվեն, և օգտագործեք var միայն այն դեպքում, եթե ձեզ անհրաժեշտ է աջակցել ավելի հին JavaScript միջավայրերին կամ ունեք հատուկ օգտագործման դեպքեր, երբ դրա վարքագիծը ցանկալի է:

harc 6

What is the different between let ,var and const.(ո՞րն է տարբերությունը let-ի, var-ի և const-ի միջև:)

Let-ի, var-ի և const-ի միջև JavaScript-ում հիմնական տարբերությունները կայանում են դրանց շրջանակի կանոնների, բարձրացման վարքագծի և վերանշանակման հնարավորությունների մեջ. Շրջանակներ. var. var-ով հայտարարված փոփոխականները ֆունկցիայի շրջանակով են: Դրանք հասանելի են ցանկացած վայրում այն գործառույթի շրջանակում, որով դրանք հայտարարված են՝ անկախ բլոկի սահմաններից: let. let-ով հայտարարված փոփոխականները շրջափակված են: Դրանք հասանելի են միայն բլոկի ներսում (փակված գանգուր փակագծերով), որտեղ դրանք հայտարարված են: const. const-ով հայտարարված փոփոխականները նույնպես շրջափակված են, ինչպես let-ը: Բարձրացում: var. Var-ով հայտարարված փոփոխականները բարձրացվում են իրենց գործառույթի շրջանակի կամ գլոբալ շրջանակի վերևում, ինչը նշանակում է, որ դրանք կարող են մուտք գործել նախքան դրանք հայտարարվելը: let. let-ով հայտարարված փոփոխականները բարձրացվում են իրենց բլոկի շրջանակի վերևում, բայց դրանք չեն սկզբնավորվում (նրանց սկզբնական արժեքը մնում է չսահմանված), մինչև այն տողը, որտեղ դրանք հայտարարված են, չի կատարվում: const. const-ով հայտարարված փոփոխականները նույնպես բարձրացվում են իրենց բլոկի շրջանակի վերևում, բայց ինչպես let-ը, դրանք չեն սկզբնավորվում մինչև այն տողը, որտեղ դրանք հայտարարված են, չի կատարվում: Այնուամենայնիվ, դրանք պետք է նախաստորագրվեն մի արժեքով հայտարարման պահին և չեն կարող հետագայում վերագրվել: Վերահանձնում. var: Var-ով հայտարարված փոփոխականները կարող են վերանշանակվել կամ թարմացվել: let. let-ով հայտարարված փոփոխականները կարող են վերանշանակվել կամ թարմացվել: const. const-ով հայտարարված փոփոխականները չեն կարող վերանշանակվել սկզբնավորումից հետո: Այնուամենայնիվ, const-ով հայտարարված օբյեկտների և զանգվածների համար դրանց հատկությունները կամ տարրերը կարող են փոփոխվել, բայց փոփոխականն ինքնին չի կարող վերանշանակվել։ Ահա դրանց բնորոշ օգտագործման դեպքերի ամփոփագիրը. Օգտագործեք var, երբ Ձեզ անհրաժեշտ են ֆունկցիաների շրջանակով փոփոխականներ կամ ցանկանում եք աջակցել ավելի հին JavaScript միջավայրերին: Օգտագործեք let, երբ ձեզ անհրաժեշտ են բլոկային շրջանակով փոփոխականներ, որոնք կարող են վերանշանակվել: Օգտագործեք const, երբ ցանկանում եք հայտարարել հաստատուններ կամ անփոփոխ փոփոխականներ: Օգտագործեք այն լռելյայն այն փոփոխականների համար, որոնք չեն վերահանձնվի: Ժամանակակից JavaScript կոդում, ընդհանուր առմամբ, խորհուրդ է տրվում օգտագործել let-ը և const-ը var-ի նկատմամբ՝ շնորհիվ իրենց ավելի կանխատեսելի շրջանակի վարքի և չնախատեսված վերահանձնումները կանխելու հնարավորության: Օգտագործեք const այն փոփոխականների համար, որոնք չեն վերագրվում, և թողեք այն փոփոխականների համար, որոնք պետք է վերանշանակվեն:

Explain hosting in Javascript.(բացատրել հոսթինգը Javascript-ով)

Hoisting-ը JavaScript-ի մեխանիզմ է, որտեղ փոփոխականների և ֆունկցիաների հայտարարագրերը տեղափոխվում են իրենց պարունակող շրջանակի վերև՝ կոմպիլյացիայի փուլում՝ նախքան ծածկագրի գործարկումը: Սա նշանակում է, որ անկախ նրանից, թե փոփոխականներն ու գործառույթները որտեղ են հայտարարագրված իրենց շրջանակում, դրանք վերաբերվում են այնպես, ասես դրանք հայտարարված են շրջանակի վերևում: Բարձրացման երկու հիմնական ասպեկտ կա՝ փոփոխական բարձրացում և ֆունկցիայի բարձրացում: Փոփոխական բարձրացում. Երբ փոփոխականները բարձրացվում են, միայն դրանց հայտարարագրերը (ոչ սկզբնավորումները) բարձրացվում են իրենց պարունակող շրջանակի վերևում: Եթե փոփոխականը հայտարարվում է var-ով, ապա դրա հայտարարությունը բարձրացվում է ֆունկցիայի կամ գլոբալ շրջանակի վերևում, և այն սկզբնավորվում է undefined-ով: Գործառույթների բարձրացում. Գործառույթների հայտարարագրերը նույնպես բարձրացվում են իրենց պարունակող շրջանակի վերևում, ինչը նշանակում է, որ դրանք կարող են կանչվել նախքան կոդում հայտարարագրվելը: Ի տարբերություն փոփոխականների, ֆունկցիաների հայտարարագրերը բարձրացվում են իրենց ամբողջ մարմինների հետ միասին:

harc 8

What is the DOM.(ինչ է դոմը)

DOM-ը կամ փաստաթղթի օբյեկտի մոդելը ծրագրավորման ինտերֆեյս է վեբ փաստաթղթերի համար: Այն ներկայացնում է HTML (կամ XML) փաստաթղթերի կառուցվածքը որպես ծառի նման կառուցվածք, որտեղ յուրաքանչյուր հանգույց ներկայացնում է փաստաթղթի տարր, հատկանիշ կամ տեքստային բովանդակություն: DOM-ը ծրագրերի համար հնարավորություն է տալիս դինամիկ մուտք գործել և շահարկել վեբ էջերի բովանդակությունը, կառուցվածքը և ոճը: Ահա մի քանի հիմնական կետեր DOM-ի վերաբերյալ. Ծառի կառուցվածքը. DOM-ը կազմակերպում է HTML փաստաթղթերը հիերարխիկ ծառի կառուցվածքի մեջ, որտեղ փաստաթուղթն ինքը որպես արմատային հանգույց է: Փաստաթղթի յուրաքանչյուր տարր, հատկանիշ և տեքստային հանգույց ներկայացված է որպես ծառի հանգույց: Հանգույցներ. հանգույցները DOM ծառի կառուցման բլոկներն են: Կան տարբեր տեսակի հանգույցներ, ներառյալ տարրերի հանգույցները, հատկանիշի հանգույցները, տեքստային հանգույցները և այլն, որոնցից յուրաքանչյուրը ներկայացնում է փաստաթղթի տարբեր մասեր: Անցում և մանիպուլյացիա. Նախատիպային ժառանգությունը JavaScript-ի հիմնական հասկացությունն է, որը թույլ է տալիս օբյեկտներին ժառանգել հատկություններ և մեթոդներ այլ օբյեկտներից: Ի տարբերություն Java-ի կամ C++-ի նման լեզուներում հայտնաբերված դասական ժառանգության, որտեղ դասերն օգտագործվում են օբյեկտներ ստեղծելու համար, JavaScript-ն օգտագործում է նախատիպի վրա հիմնված ժառանգություն, որտեղ օբյեկտներն ուղղակիորեն ժառանգում են այլ օբյեկտներից: Ահա թե ինչպես է աշխատում նախատիպային ժառանգությունը JavaScript-ում. Նախատիպի շղթա. JavaScript-ի յուրաքանչյուր օբյեկտ ունի թաքնված հատկություն, որը կոչվում է __proto__ (կամ [[Prototype]]՝ հստակեցման առումով), որը մատնանշում է իր նախատիպի օբյեկտը: Երբ դուք մուտք եք գործում որևէ հատկություն կամ մեթոդ օբյեկտի վրա, JavaScript-ը նախ ստուգում է՝ արդյոք այդ հատկությունը կամ մեթոդը գոյություն ունի անմիջապես օբյեկտի վրա: Եթե ոչ, ապա այն փնտրում է նախատիպի շղթան՝ հետևելով __proto__ հղմանը դեպի նախատիպ օբյեկտ և շարունակում է որոնումը, մինչև որ գտնի հատկությունը կամ հասնի շղթայի վերջը (այսինքն՝ նախատիպը զրոյական է): Նախատիպի օբյեկտ. JavaScript-ի յուրաքանչյուր ֆունկցիա ունի հատուկ հատկություն, որը կոչվում է նախատիպ, որը օբյեկտ է, որը ծառայում է որպես նախատիպ այն օբյեկտների համար, որոնք ստեղծված են այդ ֆունկցիան որպես կոնստրուկտոր օգտագործելով: Երբ դուք ստեղծում եք նոր օբյեկտ՝ օգտագործելով կոնստրուկտոր ֆունկցիան նոր հիմնաբառով, նոր օբյեկտի __proto__ հատկությունը սահմանվում է որպես կոնստրուկտոր ֆունկցիայի նախատիպի հատկություն: Կոնստրուկտորի գործառույթները. Constructor ֆունկցիաները օգտագործվում են ընդհանուր նախատիպով օբյեկտներ ստեղծելու համար: Դրանք նման են այլ լեզուների դասերին, բայց JavaScript-ում սովորական գործառույթներ են: Դուք սահմանում եք հատկություններ և մեթոդներ կոնստրուկտորի ֆունկցիայի նախատիպի օբյեկտի վրա՝ դրանք հասանելի դարձնելու այդ կոնստրուկտորով ստեղծված օբյեկտների համար: Օբյեկտների ստեղծում.
